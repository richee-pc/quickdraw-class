  • 데이터 편향이 결과에 미치는 영향을 사례로 말할 수 있다.
수업 안내

  • 1차시: 도입·핵심 개념

  • 4~5차시: Colab 코딩 실습 + 게임 완성

AI 두 얼굴

04. 생성형 vs 분류형

  • 생성형: 새로운 결과물을 만든다 (예: 미드저니)
  • 분류형: 입력을 구분해 이름을 맞힌다 (예: 퀵드로우)
  • 오늘 핵심은 분류형 AI를 만드는 것
퀵드로우란?

05. Quick, Draw!의 핵심 아이디어

  • 많은 사람이 빠르게 그린 낙서 데이터를 사용
  • 28x28 픽셀 같은 작은 이미지로도 분류 가능
  • 짧은 시간에 AI 학습 구조를 체험하기 좋음
핵심 질문

06. AI는 어떻게 그림을 맞힐까?

  • 무엇을 보고 판단하는가?
  • 왜 어떤 그림은 잘 맞히고, 어떤 그림은 틀릴까?
  • 데이터를 바꾸면 결과도 달라질까?
원리 1

07. 데이터(Data)

  • 학습의 재료
  • 데이터가 다양할수록 일반화 성능이 좋아짐
  • 데이터 부족/불균형은 오차를 키움
원리 2

08. 학습(Training)

  • 정답이 있는 데이터로 패턴을 찾는 과정
  • 모델의 파라미터(가중치)가 업데이트됨
  • 학습 횟수(Epoch)가 너무 적거나 많으면 문제 발생
원리 3

09. 모델(Model)

  • 입력을 받아 확률을 출력하는 수학적 함수
  • CNN은 이미지에서 특징을 단계적으로 추출
  • 좋은 모델 + 좋은 데이터가 함께 필요
원리 4

10. 추론(Inference)

  • 처음 보는 입력을 예측
  • 학습 때 본 패턴과 비슷한지 비교
  • 실전 성능은 추론 단계에서 확인
시각 인식

11. CNN이 보는 것

  • 초기층: 선/점/모서리
  • 중간층: 형태 조합
  • 후반층: 클래스별 특징
성능 지표

12. 정확도(Accuracy)

  • 전체 중 맞힌 비율
  • 수업에서는 빠른 비교에 유용
  • 하지만 클래스 불균형이면 오해 가능
성능 지표

13. 손실(Loss)

  • 예측이 얼마나 틀렸는지 수치화
  • 학습 중 loss는 내려가고 accuracy는 올라가길 기대
  • 두 지표를 함께 확인해야 안정적
학습 분할

14. Train / Validation / Test

  • Train: 학습용
  • Validation: 학습 중 점검
  • Test: 최종 평가(처음 보는 데이터)
과적합

15. Overfitting이란?

  • 학습 데이터만 너무 잘 맞는 상태
  • 새로운 데이터에서 성능이 떨어짐
  • 해결: 데이터 다양화, 모델 단순화, 정규화
소프트웨어 원리

16. 입력 → 처리 → 출력

  • 입력: 학생 스케치
  • 처리: 전처리 + 모델 예측
  • 출력: 클래스 확률/결과 화면
소프트웨어 원리

17. 모듈화

  • 기능을 함수/셀 단위로 분리
  • 문제 발생 시 수정 지점 찾기 쉬움
  • 협업과 재사용성 향상
소프트웨어 원리

18. 파이프라인 사고

  • 데이터 수집 → 정제 → 학습 → 평가 → 배포
  • 한 단계 오류가 다음 단계에 전파
  • 단계별 검증이 중요
소프트웨어 원리

19. 버전 관리 감각

  • 코드/파라미터 변경 이력 기록
  • 정확도 변화 원인 추적
  • 실험 노트 습관이 실력 차이를 만듦
소프트웨어 원리

20. 재현 가능성

  • 같은 코드/같은 데이터/같은 설정이면 비슷한 결과
  • 랜덤 시드로 변동성 줄이기
  • 실험 결과를 신뢰할 수 있게 함
소프트웨어 원리

21. UI와 로직 분리

  • 화면(Gradio/Streamlit)과 모델 로직 분리
  • 디자인 변경 시 모델 영향 최소화
  • 테스트 범위가 명확해짐
소프트웨어 원리

22. 에러 처리

  • 예: 빈 입력일 때 예외 처리
  • 친절한 메시지로 사용자 경험 향상
  • 수업 중 진행 중단 방지
소프트웨어 원리

23. 성능/속도 균형

  • 모델을 복잡하게 하면 정확도↑ 가능
  • 하지만 학습 시간↑, 실습 몰입도↓
  • 수업에 맞는 균형이 중요
소프트웨어 원리

24. 데이터 직렬화

  • JSON/NPY로 저장해 재사용
  • 수집기 데이터와 Colab 연결
  • 파일 포맷 이해는 실무 기본
소프트웨어 원리

25. API 개념

  • 요청(Request)과 응답(Response)
  • 수집기 제출 데이터가 서버로 이동
  • classId/objectName으로 데이터 구분
코딩 개념

26. 변수(Variable)

  • 값을 저장하는 이름표
  • 예: categories, EPOCHS
  • 좋은 변수명은 이해도를 높임
코딩 개념

27. 리스트(List)

  • 여러 값을 순서대로 보관
  • 예: 클래스 이름 목록
  • 반복문과 함께 자주 사용
코딩 개념

28. 딕셔너리(Dictionary)

  • 키-값 구조
  • 예: {label: probability}
  • 예측 결과 표현에 유용
코딩 개념

29. 조건문(if)

  • 상황에 따라 다른 동작
  • 예: 값이 None이면 에러 안내
  • 안전한 코드의 핵심
코딩 개념

30. 반복문(for)

  • 같은 작업을 자동 반복
  • 예: 클래스별 데이터 로딩
  • 코드 양을 크게 줄여줌
코딩 개념

31. 함수(function)

  • 작업 묶음
  • 예: add_conv_block(), predict()
  • 재사용성과 가독성 향상
코딩 개념

32. 라이브러리 import

  • 이미 만들어진 도구 사용
  • numpy, tensorflow, gradio
  • 도구를 이해하고 적재적소에 사용
코딩 개념

33. 배열/텐서 shape

  • 데이터 모양이 다르면 오류
  • 이미지 shape: (N, 28, 28, 1)
  • shape 확인 습관이 중요
코딩 개념

34. 정규화(normalization)

  • 0~255를 0~1로 스케일
  • 학습 안정성과 수렴 속도 개선
  • 전처리의 핵심 단계
코딩 개념

35. 하이퍼파라미터

  • EPOCHS, batch_size, learning 관련 설정
  • 성능과 속도에 큰 영향
  • 실험적으로 조정
코딩 개념

36. 함수형 사고

  • 입력과 출력을 명확히
  • 중간 단계 점검(print/shape)
  • 디버깅 시간을 줄임
코딩 개념

37. 예외와 메시지

  • 오류를 빨리 이해하게 안내
  • 학생 실습에서는 특히 중요
  • 힌트가 좋은 수업을 만든다
실습 준비

38. Colab 환경 체크

  • 런타임 연결 확인
  • 필수 패키지 설치
  • 셀은 위에서 아래 순서대로 실행
실습 1

39. 카테고리 설정

  • 3~5개 클래스를 영어로 선택
  • 서로 너무 비슷한 클래스는 초반엔 피하기
  • 예: cat, fish, apple, house
실습 2

40. 데이터 로딩

  • 클래스별 .npy 파일 다운로드
  • N_PER_CLASS로 장수 조절
  • 로딩 후 클래스별 개수 출력 확인
실습 3

41. 수집기 데이터 연결(선택)

  • USE_COLLECTOR_DATA=True
  • JSON 업로드 후 classes/X/y 업데이트
  • 수집기와 본 학습이 연결됨
실습 4

42. 데이터 시각화

  • 랜덤 샘플 확인
  • 클래스 라벨이 의도대로인지 점검
  • 깨진 데이터/이상치 확인
실습 5

43. 코드 작성 포인트 A

  • DIVISOR 값 채우기
  • CHANNELS 값 채우기
  • shape 출력으로 전처리 결과 검증
실습 6

44. 코드 작성 포인트 B

  • ACTIVATION / FILTERS 설정
  • add_conv_block() TODO 완성
  • 모델 구조를 직접 구성
실습 7

45. 학습 실행

  • 학습 로그에서 loss/accuracy 관찰
  • Validation 지표 함께 보기
  • 너무 오래 걸리면 파라미터 조정
실습 8

46. 평가

  • test 정확도 확인
  • 그래프에서 학습 곡선 읽기
  • 개선 아이디어 기록
실습 9

47. Gradio 게임 배포

  • Sketchpad 입력
  • 예측 확률 상위 3개 출력
  • 친구들과 링크 공유 테스트
실습 10

48. Midjourney 아트 연결(선택)

  • 정답 라벨별 이미지 업로드
  • 예측 1등 라벨 아트 표시
  • 게임 몰입감 상승
실습 11

49. 오류 해결 가이드

  • NameError: 위 셀 실행 여부 확인
  • shape 오류: reshape/채널 점검
  • 예측 이상: 전처리/정규화 확인
실습 12

50. 속도 최적화

  • 클래스 수 줄이기
  • N_PER_CLASS 조정
  • EPOCHS와 정확도 균형 맞추기
실습 13

51. 편향 실험

  • 비슷한 그림만 넣고 성능 확인
  • 다양한 그림 추가 후 재학습
  • 전/후 성능 변화 기록
실습 14

52. 팀 미션

  • 각 팀 테마 클래스 구성
  • 가장 높은 정확도 팀
  • 가장 창의적인 아트 팀
실습 15

53. 체크포인트

  • 코드 TODO 모두 완료했는가
  • 예측이 실제로 동작하는가
  • 설명 가능한 결과를 얻었는가
윤리와 책임

54. 데이터 편향의 사회적 의미

  • 데이터가 치우치면 사용자에게 불리할 수 있음
  • 공정성과 다양성 고려는 필수
  • AI 성능뿐 아니라 영향까지 생각하기
윤리와 책임

55. 개인정보/저작권

  • 필요 이상 정보 수집 금지
  • 사용 이미지의 권리 확인
  • 출처 표시 습관화
윤리와 책임

56. 투명성

  • 모델이 완벽하지 않음을 명시
  • 오류 가능성과 한계 공유
  • 책임 있는 발표 태도
발표 준비

57. 발표 구조 템플릿

  • 문제 정의: 무엇을 만들었나
  • 방법: 어떤 데이터/코드 사용
  • 결과: 정확도/오류/개선점
발표 준비

58. 데모 체크

  • 인터넷/링크 확인
  • 마이크/화면 준비
  • 질문 예상 답변 준비
발표 준비

59. 피드백 기준

  • 개념 이해도
  • 코드 작성 참여도
  • 실험/개선 시도
  • 설명의 명확성
정리

60. 오늘의 핵심 5문장

  • AI 성능은 데이터의 질에 크게 좌우된다.
  • 코드는 작은 TODO부터 직접 고치며 배운다.
  • 편향을 줄이려면 다양한 데이터가 필요하다.
  • 모델 정확도는 측정하고 비교해야 한다.
  • 기술은 결과뿐 아니라 영향까지 생각해야 한다.
